# DICES_TZ

## Суть идеи
Реализовать броски дайсов так, чтобы:
- если окно карты **не открыто** — результат считается текущей логикой RNG (быстро и стабильно);
- если окно карты **открыто** — результат берется из 3D-симуляции броска кубиков на карте;
- результат всегда отображается в окне дайсов в едином формате.

Цель: сделать эффектный визуальный бросок без потери надежности и без зависаний UI.

## Архитектура

### 1) DiceService (домен)
Единая точка входа `roll(request, mode)`:
- `mode=random` — обычный расчет случайных значений;
- `mode=physics` — результат приходит из 3D-симуляции.

Возвращает унифицированный `RollResult` для окна дайсов.

### 2) DicePhysicsService (инфраструктура)
Отвечает за 3D-бросок:
- спавн кубиков;
- шаг симуляции/ожидание стабилизации;
- вычисление верхней грани у каждого куба;
- формирование результата и отправка в DiceService.

### 3) MapDiceOverlay (QML слой)
Отдельный слой поверх карты (`View3D + physics`):
- визуализация падения/качения/остановки кубиков;
- не смешивается с логикой инструментов карты.

### 4) DiceController
Не считает броски напрямую, а отправляет запрос в сервис и получает готовый `RollResult`.

### 5) EventBus
Событийная шина:
- `dice.roll_requested`
- `dice.roll_completed`

## Как определять выпавшее значение
Для каждого типа куба хранить таблицу соответствия:
- `локальная нормаль грани -> значение`.

После остановки куба:
1. Берем его world rotation.
2. Поворачиваем нормали граней в world space.
3. Считаем `dot(faceNormal, worldUp)`.
4. Грань с максимальным dot = верхняя, ее значение = результат.

## Надежность
- Таймаут симуляции (например 5 сек).
- Если кубики не стабилизировались — fallback на RNG.
- Приложение не должно зависать, UI должен оставаться отзывчивым.

## План реализации

1. Ввести общий контракт `RollRequest/RollResult`.
2. Перевести DiceController на сервисный вызов и события `roll_requested/roll_completed`.
3. Добавить режимы в DiceService: `random` и `physics`.
4. Добавить `MapDiceOverlay.qml` в окно карты (отдельный слой).
5. Реализовать `DicePhysicsService` для одного типа куба (PoC на d6).
6. Реализовать определение верхней грани через нормали.
7. Расширить на d4/d8/d10/d12/d20.
8. Добавить d20 advantage/disadvantage в physics-режиме.
9. Реализовать d100 как два d10 (десятки + единицы).
10. Добавить таймауты, fallback, блокировку кнопок во время симуляции, тесты.
